Interactive Fluid Simulation Checkpoint 1

In this milestone, we looked into various frameworks and ultimately settled on using three.js and React for our water simulation project. Thus far, the particle system is set up with the following physics properties: pressure, viscosity, gravity, and surface tension.

The following is a small gif of a fluid simulation running

We also attempted to use our own custom Blinn-Phong shader as opposed to using the built-in one. The vertex shader and fragment shader for our Blinn-Phong shader were working as expected for individual particles (spheres). We did have bugs in our implementation of the shader, causing vertices to be updated to a single point. This has yet to be fixed, but we believe it is due to using a combined mesh with all the particles recieving a reference to the same shader.

Our biggest issue, however, was the lack of performance in working with rendering the different particles on to the screen. This is because on each frame update we had to recompute the forces required and update every single particle The first optimization we decided to do was combine multiple mesh objects into a single mesh to try optimize. We also attempted to optimize our code by offloading computation logic to a faster language like golang to perform the simulation. However, we weren't able to get the transfer to work for this checkpoint.

In terms of relative performance hit, we are able to render 100 particles with about 45-60 fps. However, when we scale up to a 1000 particles, we easily see a drop in performance down to 1-3 fps. This probably due to our single threaded javascript implementation, which we haven't been able to fix for the checkpoint.

With all the performance issues we've been seeing, we've been considering switching over to a faster webgl system either in c,c++ in order to render larger water simulations. Since we believe that a small set of 100 particles is not enough.

Since we have implemented the general force logic and the updates with implicit euler, we believe that we are still on track even if we have the change frameworks. We hope in a possible new framework to get user interactions and realstic shaders working.

To run the code, cd into the src directory and run `npm run start`

Interactive Fluid Simulation Proposal

Summary

We implement a fluid simulation that accounts for external forces such as user interaction. This allows for more realistic fluid simulation. We extend our work based on the paper.

Team Members 

Vikranth Srivatsa, Shunyuet (Benjamin) Cheung, Bhuvan Basireddy, Kevin Lo

Problem Description

        A lot of past projects in CS184 dealt with fluid simulations but do not account for the external forces such as user interaction in a realistic way. We try to use realistic fluid simulations that allow for user interactions and swirling the water. This can allow for simulations of water that have whirlpools and such.

Goals and Deliverables

        We plan on producing realistic simulations by modeling fluids with particles. The math behind the scenes would include formulas that consider pressure, surface tension, viscosity, and external forces. An example of what we hope to achieve is pouring water into a glass, which can be found in the paper listed in the “Resources” section of this proposal.

Here the water is being poured into a glass.

  Here the water is sloshing and settling.

Since our hope is that we want to create an interactive demo, we are trying to either render a website demo and use the mouse in order to display the interaction. In terms of measuring quality, we hope to measure the quality using a visual test of how the liquid simulation looks. It’s hard to measure quantitatively so we hope that the graphics demo is a good way to display the simulation.

If things go ahead of schedule, we hope to simulate other liquids and fluids. It would also be interesting to look into seeing when the container itself moves over the simulation time. It would also be interesting to see how ice is acting in the water.

Schedule:

Week 1:

  • Setup basic fluid simulations with a lot of particles using navier-stokes equations

Week 2:

  • Implement external forces that allow for tornados and some user interactions

Week 3:

  • Building out demo setup and add support for user interaction to exact forces

Week 4:

  • We hope to have a working demo with the realistic fluid simulation.
